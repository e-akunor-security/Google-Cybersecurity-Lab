==========================================================
        SECURITY AUTOMATION: BASH SCRIPTING
==========================================================

1. THE SHEBANG ( #!/bin/bash ):
   Tells the computer this is a script, not just text.

2. VARIABLES ( LOG_FILE="/path" ):
   Saves data so you don't have to type it over and over.

3. COMMAND SUBSTITUTION ( FAIL_COUNT=$(...) ):
   Saves the *result* of a command into a variable.

4. IF/THEN LOGIC:
   The "Brain" of the script. 
   If [ $COUNT -gt 5 ] -> Then ALERT.

5. PERMISSIONS ( chmod +x ):
   The command that turns a text file into a tool.
==========================================================

----------------------------------------------------------------
PHASE 13: CRON AUTOMATION (THE HEARTBEAT)
----------------------------------------------------------------
* Command: crontab -e
* Syntax: [Min] [Hour] [Day] [Month] [Weekday] [Command]
* Example: * * * * * (Runs every single minute)
* Why it matters: This turns a "Script" into a "Service." 
  A real SOC analyst automates repetitive tasks so they can 
  focus on high-level threat hunting.
